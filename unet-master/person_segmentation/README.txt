                    人物分割项目 - 使用说明
================================================================================

项目名称：Person Segmentation using U-Net
创建日期：2026-02-17
基于：U-Net 医学图像分割项目

================================================================================
一、项目概述
================================================================================

本项目基于 U-Net 网络实现人物图像分割，可以将人物从背景中分离出来。

主要特点：
- 支持 RGB 彩色图像输入
- 自动人物分割
- 简单易用的训练和预测流程

三、快速开始
================================================================================

【步骤 1：准备训练数据】

方法 A：使用公开数据集（推荐）
  1. 下载人物分割数据集：
     - Supervisely Person Dataset
     - Pascal VOC 2012 (Person class)
     - LIP Dataset
  
  2. 将数据放到对应目录：
     - 人物照片 -> dataset/train/image/
     - 人物标注 -> dataset/train/label/

方法 B：自己标注数据
  1. 收集人物照片（建议 100+ 张）
  2. 使用标注工具：
     - LabelMe: https://github.com/wkentaro/labelme
     - VGG Image Annotator: http://www.robots.ox.ac.uk/~vgg/software/via/
  3. 标注人物轮廓（白色=人物，黑色=背景）
  4. 保存到 dataset/train/label/

【步骤 2：训练模型】

在 person_segmentation 目录下运行：
  python train_person.py

训练时间：
  - 100 张图：约 30-60 分钟
  - 500 张图：约 1-2 小时
  - 1000 张图：约 2-4 小时

【步骤 3：预测新图像】

1. 将测试图像放到 dataset/test/ 目录
   - 命名格式：0.jpg, 1.jpg, 2.jpg, ... 或 0.png, 1.png, ...
   
2. 运行预测：
   python predict_person.py

3. 查看结果：
   - 打开 dataset/test/ 目录
   - 对比原图和预测结果

================================================================================
四、数据格式要求
================================================================================

【训练图像】
- 格式：JPG 或 PNG
- 颜色：RGB 彩色图
- 尺寸：任意（会自动调整到 256x256）
- 内容：包含人物的清晰照片

【训练标注】
- 格式：PNG
- 颜色：灰度图
- 尺寸：与对应训练图像相同
- 内容：
  * 白色（255）= 人物区域
  * 黑色（0）= 背景区域

【测试图像】
- 格式：JPG 或 PNG
- 颜色：RGB 彩色图
- 尺寸：任意
- 命名：0.jpg, 1.jpg, ... 或 0.png, 1.png, ...

================================================================================
五、训练参数
================================================================================

【网络参数】
- 输入尺寸：256 x 256 x 3 (RGB)
- 输出尺寸：256 x 256 x 1 (分割mask)

【训练参数】
- Batch size: 2
- Steps per epoch: 300
- Epochs: 1 (可修改为 10, 20, ...)
- Learning rate: 0.0001
- Optimizer: Adam
- Loss: Binary Crossentropy

【数据增强】
- 旋转：±20%
- 平移：±5%
- 剪切：±5%
- 缩放：±5%
- 水平翻转：是

================================================================================
六、调整训练参数
================================================================================

【增加训练轮数】
编辑 train_person.py，找到：
  model.fit(myGene, steps_per_epoch=300, epochs=1, ...)

修改为：
  model.fit(myGene, steps_per_epoch=300, epochs=10, ...)

【调整学习率】
编辑 model_rgb.py，找到：
  model.compile(optimizer=Adam(learning_rate=1e-4), ...)

修改为：
  model.compile(optimizer=Adam(learning_rate=1e-3), ...)  # 更快
  或
  model.compile(optimizer=Adam(learning_rate=1e-5), ...)  # 更稳定

【调整 batch size】
编辑 train_person.py，找到：
  myGene = trainGenerator_rgb(2, ...)

修改为：
  myGene = trainGenerator_rgb(4, ...)  # 更快，但需要更多内存
  或
  myGene = trainGenerator_rgb(1, ...)  # 更慢，但内存占用少

七、常见问题
================================================================================

【问题 1：找不到训练图像】
解决：
  - 确保 dataset/train/image/ 目录中有图像
  - 检查图像格式是否正确（.jpg 或 .png）

【问题 2：找不到训练标注】
解决：
  - 确保 dataset/train/label/ 目录中有标注
  - 标注文件名必须与图像文件名相同
  - 标注必须是灰度图（黑白）

【问题 3：训练效果差】
解决：
  - 增加训练数据（至少 100 张）
  - 增加训练轮数（epochs=10 或更多）
  - 确保标注质量高
  - 调整学习率

【问题 4：预测结果全黑或全白】
解决：
  - 检查模型是否正确加载
  - 尝试重新训练模型
  - 调整阈值（在 predict_person.py 中）

【问题 5：内存不足】
解决：
  - 减小 batch size
  - 减小图像尺寸
  - 使用 GPU

十一、注意事项
================================================================================

1. 训练数据质量很重要
   - 标注要准确
   - 图像要清晰
   - 数据量要足够

2. 训练时间取决于数据量
   - 数据越多，时间越长
   - 可以先用少量数据测试

3. GPU 可以加速训练
   - CPU: 几小时
   - GPU: 几十分钟

4. 预测结果可以后处理
   - 形态学操作（腐蚀、膨胀）
   - 平滑边缘
   - 去除小区域
